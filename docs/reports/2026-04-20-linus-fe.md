## FOR USER — E2E-Prod-Rate-Limit

Die Prod-E2E-Runs schicken echte SMTP-Mails über IONOS (46/Run bei 23 Tests × 2 Mails). Das hat heute den Funnel mid-run rate-limited. Drei Optionen für die nächste Session:

| # | Option | Pro | Contra |
|---|---|---|---|
| a | **Prod-E2E nur nightly via GitHub Actions**, sequenziell mit 2s-Delay pro Test | Ehrlichster End-to-End-Pfad, IONOS hält durch | Kein sofort-Feedback für User-Re-Deploys |
| b | `SMTP_TEST_BYPASS` Env-Var die sendFormEmail in Prod zu No-Op macht wenn Bypass-Header gesetzt ist | Schnelle Prod-Runs | Bypass-Header darf nicht leaken, minimale Attack-Surface |
| c | Akzeptieren dass Prod-E2E flaky ist, Fokus auf lokale 23/23 als CI-Gate | Null Code-Aufwand | Kein Confidence für SMTP-Pfad |

Linus-Empfehlung: **a** (nightly), mit optionaler `--grep` Ausnahme für Smoke-Only-Run on-demand.
